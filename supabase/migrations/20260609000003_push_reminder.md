# ⚠️  Run this SQL in the Supabase Dashboard to fix the 400 errors

The local migration `20260323070000_update_house_to_villa.sql` was never pushed
to the live database.  Until you run it, any filter on `type=eq.villa` returns
HTTP 400 because the PostgreSQL enum still has `'house'` as a value.

## Option A — Supabase CLI (recommended)

```bash
npx supabase db push
```

## Option B — Supabase Dashboard SQL Editor

Go to: https://supabase.com/dashboard/project/hetaveowlxcjuxbtckqt/sql/new

Paste and run:

```sql
-- Rename the enum value 'house' → 'villa'
-- (safe to re-run: IF VALUE EXISTS guard not needed in PG 15+)
DO $$
BEGIN
  IF EXISTS (
    SELECT 1 FROM pg_enum e
    JOIN pg_type t ON t.oid = e.enumtypid
    WHERE t.typname = 'property_type'
      AND e.enumlabel = 'house'
  ) THEN
    ALTER TYPE property_type RENAME VALUE 'house' TO 'villa';
    UPDATE public.properties SET type = 'villa' WHERE type = 'house';
  END IF;
END $$;
```
